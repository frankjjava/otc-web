Code Contributors


Hope you liked the OTC framework. If you find a feature that is missing and would like to contribute code, or would like to work on the future versions with features listed in the "Download, Releases" page", that's great!

OTC Framework adopts the fork-n-pull approach. Just fork the repository to your GitHub account and create a new branch for each feature or bug-fix. Once you are done with your changes, create a pull request for merging with the OTC Framework repo.

When doing changes, please follow the below best practices:

  1. Before getting started, discuss new features / bug fix you’d like to implement at the Discussions link given in the previous page
  2. Create one pull request per feature / bug fix
  3. Configure git to automatically manage line endings as described in Configuring Git to handle line endings
  4. Format code to keep it the same as the existing OTC source-code. Please set the formatter template in your IDEs appropriately
  5. Start your commit messages with "#", e.g. "#777 Support for nested generics"
  6. Please provide appropriate OTCS test case script files
  7. Update the reference documentation if required on the website repo at the link given above
