Section 11: OTC Configuration

In this section, we will look at configuring the OTC Engine.

We already got familiar with "${OTC_HOME}" folder in the 'Getting Started' section. But now lets look at the folder structure in detail -



  1. The "config" folder:
    This is the folder where the configuration files are stored.

    There are two configuration files "otc.properties" and "log.properties" which we need to be concerned about. These property files consist of the OTC engine configurations and logging configurations.

    • The "otc.properties" file:

      The OTC configurations are kept in this file. Both the OTCS editor and the OTC Engine loads the configuration defined in this file. It has only one property which we should be concerned about and that is - 

      executor.packages.filter= --enter package-names --

      Your DTO classes will belong to a package. Its members may consist of classes from other libraries and belonging to other packages. But you may want to filter classes of only some desired packages. For example, in the Editor you may want to filter only the classes of specific packages in the source-tree and the target-tree. In such a case you may set this property appropriately. For multiple packages, put them in separate lines as shown here – 
      executor.packages.filter=com.athena.airlines.dto \
            com.kronos.airlines.dto

    • The "log.properties" file:
      The logging configurations are kept in the log.properties file which are used in the "logback-included.xml" file present in the same folder. Please take a look at these files for logging configurations.

  2. The "lib" folder:
    This folder is the location where the jar files containing the DTO classes are stored which are used by the OTCS editor.

  3. The "logs" folder:
    This folder is the location where the the Editor and the Engine stores application logs.

  4. The "otc-pretest" folder:
    This folder is used as a store for all "*.otcs" OTCS files meant for unit-testing.

  5. The "otc-unittest" folder:
    This folder is where the OTCS Compiler looks for the OTCS files when the "compile()" method on the compiler is invoked. When you want to test multiple OTCS files, test them one by one by copying them from the "pretest" folder to this folder. A directory structure under this folder is translated to a package-name in the generated source code files. For example, a folder structure such as "org/otcl" as shown here will create the generated source code under the package "org.otcl".

  6. The "otc-pretest" folder:
    This folder is where the already tested OTCS files are kept. After testing the OTCS files, move them to this folder in order to avoid repeat compilations of the already tested OTCS files on every test iteration.

  7. The "src" folder:
    This folder is the location where the compiler stores the generated source code files.

  8. The "target" folder:
    This folder is the location where the compiler stores the compiled class files.

  9. The "test-samples" folder:
    This folder consist of the Kronos-passenger-map.xml file used to build an object of KronosAirlinePassenger which is used as the source object in this course.
