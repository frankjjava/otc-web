OTC in 1 minute


What you would love about OTC is, there is no big learning curve. Thats because you don't have to code anything new, but rather only learn how to use the OTC script UI and compile it - no interfaces, no annotations, no lengthy manual configurations.

Consider an object tree as shown below. On the left side is source object tree, and, on the right side is the target object tree. And the objective is to copy values from the source to the target.

Observe the object-path to the "value" field on the source-tree and the object-path to the "ptc" field on the target-tree.



Let me represent both these object paths as shown below. 
from:
  objectPath: travelers.traveler[*].anonymousTravelerMap[*,*]<V>.ptc.value
to:
  objectPath: passengerList.passenger[*].ptc

In the above OTC-Script fragment, the token suffixed with a "[*]" represents a list or an array. And the token suffixed with a "[*,*]<V>" represents a value in a key-value pair of a dictionary entry (map-entry).


Now here we go with the "OTC in 1 minute" -

  1. Fire up the OTC Script editor (Refer the section on "OTCS Editor")
  2. Navigate and make your selections.
  3. Click on "Add OTCS command"
  4. Repeat steps 2 and 3 for all the fields you wish to copy
  5. Click on "Create File" and save the file under "${OTC_HOME}/unittest" folder
  6. Execute the "OtcTest.java" class in the "otc-test" project. (Refer the section on "Test Project" for more details).

There you go - we are done !! All the object-conversion source-code is generated and executed in a jiffy !!!

If you navigate to the "${OTC_HOME}/src" folder, you will find all the generated source code files.

On the whole from start-to-finish It should have taken you not more than 10 or 15 minutes to complete the above steps.
