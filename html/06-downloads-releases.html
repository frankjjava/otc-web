Releases

  1. Version 2.6 - current version
    OTCS script parameter name change in couple of them.

  2. Version 2.7
    Support for Optional container type.

  3. Version 2.8
    Support for nested generics.

    Support for JAXB Object-factory parameters in OTCS scripts and implementation.

  4. Version 3.0
    Grouping: When we are copying multiple fields from one composite object in the source to another composite object in the target, only the leaf-nodes (leaf field-names) change in the object-paths while the remaining is the same. Presently OTC generates distinct source files for each OTCS command. The objective is this:- during compilation, group the OTCS commands where the source-object-path and the target-object-path across multiple OTCS commands differs only in the leaf-node. This will reduce the number of source files generated to 1 for the group instead of 1 source file per OTCS command in the group.

  5. Version 3.1
    Multiple Anchors: For collection correlations which is explained in detail in the section "Anchors", presently anchoring is allowed only on the target-object-path and moreover, only a single Anchor is allowed. For multiple co-relations, presently multiple "*.otcs" files has to be created.

    The objective of this version is to allow multiple correlations with multiple Anchor support in a single script in situations were collection-counts in both the source-object-path and the target-object-path are greater than 2.
